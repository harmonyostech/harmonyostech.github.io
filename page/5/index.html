2. Does HarmonyOS provide an API to parse the broadcast data segment of Bluetooth devices?

Similar to ScanRecord for Bluetooth broadcast data sources: Start BLE scanning via startBLEScan, and obtain the data information from the scanned ScanResult. The requirement is to parse the broadcast data segment information of Bluetooth devices, such as advertisData, advertisServiceUUIDs, localName, serviceData, etc. However, the official API documentation does not provide detailed explanations for these data fields.

Broadcast Packet Data Description:

https://gitee.com/openharmony/docs/blob/master/zh-cn/application-dev/reference/apis-connectivity-kit/js-apis-bluetooth-ble.md#advertisedata

AdvertiseData

Describes the content of a BLE broadcast packet, with a length of 31 bytes.

  • Atomic Service API: This interface supports use in atomic services starting from API version 12.
  • System Capability: SystemCapability.Communication.Bluetooth.Core.
Name Type Readable Writable Description
serviceUuids Array Yes Yes Represents the list of service UUIDs to be broadcast.
manufactureData Array<ManufactureData> Yes Yes Represents the list of manufacturer information to be broadcast.
serviceData Array<ServiceData> Yes Yes Represents the list of service data to be broadcast.
includeDeviceName boolean Yes Yes Indicates whether to include the device name (optional parameter). true means inclusion, false or unspecified means exclusion. Note that the broadcast packet length must not exceed 31 bytes when including the device name.

Currently, no interface is provided to parse broadcast packets, so developers need to parse them manually.

3. Does it support cross-file namespace merging?

In the shared package development scenario, the goal is to place all exported content (including functions, classes, interfaces, constants, enums) under a single namespace. Since the content is extensive, it is distributed across multiple .ets files.

Attempted solutions:

  1. Adding namespaces to all .ets files: Cross-file namespaces are not automatically merged, so only content from a specific file can be exported.
  2. Re-exporting content defined in other .ets files within a namespace: Results in an error indicating that type definitions cannot be exported within a namespace.
  3. Defining all content under the current namespace in one file and referencing it from other files: Leads to circular dependencies, making unit tests unexecutable.

Solutions 1 and 3 are currently unfeasible. For Solution 2, the implementation steps are as follows:

1. Are user preferences and KVStore both asynchronously saved and read? Is there a synchronous reading method?

Are both user preferences and ohos.data.distributedKVStore (distributed key-value database) asynchronous for setting and getting values? The emulator feels quite slow (after setting, getting). Is there another KV data storage method that supports synchronization?

User preferences and KVStore are both asynchronously saved and read, and no synchronous operation APIs are provided. PersistentStorage supports synchronous KV data storage. Persistent variables should preferably be data smaller than 2KB, and large amounts of data should not be persisted because PersistentStorage’s disk writing operations are synchronous. Massive local reading and writing of data will be executed synchronously in the UI thread, affecting UI rendering performance. If developers need to store large amounts of data, it is recommended to use database APIs.

3. HarmonyOS AES decryption issues?

AES encryption and decryption are required in the app, using the Crypto Architecture Kit. The encryption and decryption code differences are not significant, mostly copied from official documentation. Encryption works, but decryption errors occur at the decoder.doFinal() position.

Refer to the following demo: 

import cryptoFramework from '@ohos.security.cryptoFramework';
import buffer from '@ohos.buffer';

function genIvParamsSpec() {
  let dataIv = convertStringToUint8Array('8NONwyJtHesysWpM')!; // new Uint8Array(arr);
  let ivBlob: cryptoFramework.DataBlob = { data: dataIv };
  let ivParamsSpec: cryptoFramework.IvParamsSpec = {
    algName: "IvParamsSpec",
    iv: ivBlob
  };
  return ivParamsSpec;
}

// Encrypt message
async function encryptMessagePromise(symKey: cryptoFramework.SymKey, plainText: cryptoFramework.DataBlob) {
  let cipher = cryptoFramework.createCipher('AES128|CBC|PKCS7');
  let iv = genIvParamsSpec();
  await cipher.init(cryptoFramework.CryptoMode.ENCRYPT_MODE, symKey, iv);
  let cipherData = await cipher.doFinal(plainText);
  return cipherData;
}

// Decrypt message
async function decryptMessagePromise(symKey: cryptoFramework.SymKey, cipherText: cryptoFramework.DataBlob) {
  let decoder = cryptoFramework.createCipher('AES128|CBC|PKCS7');
  let iv = genIvParamsSpec();
  await decoder.init(cryptoFramework.CryptoMode.DECRYPT_MODE, symKey, iv);
  let decryptData = await decoder.doFinal(cipherText);
  return decryptData;
}

async function genSymKeyByData(symKeyData: Uint8Array) {
  let symKeyBlob: cryptoFramework.DataBlob = { data: symKeyData };
  let aesGenerator = cryptoFramework.createSymKeyGenerator('AES128');
  let symKey = await aesGenerator.convertKey(symKeyBlob);
  console.info('convertKey success');
  return symKey;
}

async function aesCBC() {
  let keyData = convertStringToUint8Array('sgg45747ss223455')!;
  let symKey = await genSymKeyByData(keyData);
  let message = "This is a test,just";
  let plainText: cryptoFramework.DataBlob = { 
    data: new Uint8Array(buffer.from(message, 'utf-8').buffer) 
  };
  let encryptText = await encryptMessagePromise(symKey, plainText);
  let decryptText = await decryptMessagePromise(symKey, encryptText);
  if (plainText.data.toString() === decryptText.data.toString()) {
    console.info('decrypt ok');
    console.info('decrypt plainText: ' + buffer.from(decryptText.data).toString());
  }
}

1. HarmonyOS Bluetooth Key Module?

After setting the setCharacteristicChangeNotification in the Bluetooth key module, the characteristic value change cannot be monitored in the BLECharacteristicChange subscription event.
Steps:

  1. Call the setCharacteristicChangeNotification interface to set the notify property of the characteristic to true.
  2. Call the writeDescriptorValue interface to set the descriptor value of the characteristic to {0x01, 0x00}.
  3. Call the writeCharacteristicValue interface to successfully send 7 - byte data to the master device.
  4. The returned data cannot be monitored in the BLECharacteristicChange callback.

If the device itself supports data return after writing data, it is necessary to check whether the interface method of writeCharacteristicValue is inconsistent with the service of the device that will return data. You can refer to the following: https://gitee.com/openharmony/docs/blob/master/zh - cn/application - dev/reference/apis - connectivity - kit/js - apis - bluetooth - ble.md#blecharacteristic

2. Error when getting UDID via hdc shell bm get --udid in HarmonyOS?

The command hdc shell bm get --udid returns an error: [Fail]ExecuteCommand need connect-key? How to resolve this?

UDID acquisition method:

  1. Open “Settings > About Phone”, tap the version number multiple times to enable Developer Mode.
  2. Open “Settings > System & Updates”, find “Developer Options” at the bottom, and enable “USB Debugging”.
  3. After connecting the phone, open the command line tool, navigate to the HDC directory (typically: HarmonyOS SDK installation directory/toolchains/{version}), and run hdc shell bm get --udid to get the device UDID.

Troubleshooting steps:

  1. Check if both the emulator and real device are connected simultaneously. If so, close one before using the hdc command.
  2. Reconnect the phone and retry, ensuring the phone is recognized in DevEco Studio.
  3. Run hdc list targets to check if device information is obtained. If not, check for hdc processes in Task Manager. If present, kill the process with hdc kill -r and restart the hdc service. If issues persist, test with another phone.

When multiple devices are connected: Specify the target device with hdc -t xxx shell.
Example:

4. How to receive Map-type data from a JSON object using new Map()?

An Interface object receives data parsed from a JSON string as follows: 

export interface YNActionMode {  
  id?: string;   
  params?: Map<string, Object>;   
  sortFieldMap?: Map<string, Object>;   
  target?: string;
}

The params value exists, but it is actually an Object rather than a Map, so for of or forEach cannot be used for traversal. To copy params into a new Map: 

let newParams = new Map<string, Object>();

Direct assignment newParams = params converts newParams to an Object, causing data modification to affect the original params. Instead, traverse params to assign values:

Use Object.keys() combined with forEach() to traverse the object: 

const obj = {a: 1, b: 2, c: 3};
Object.keys(obj).forEach(key => { 
  console.log(key + ': ' + obj[key]);
});

5. How to set up Hyper-V on Windows 10 Home Edition?

The Hyper-V option is missing in “Control Panel > Programs > Turn Windows features on or off”. How to resolve this?

Windows 10 Home Edition can install and run Hyper-V. Follow these steps to add the feature:

  1. Verify system support: Open Windows PowerShell, run systeminfo. Check the processor requirements—if “Yes” appears four times, the system supports Hyper-V.
  2. Add the feature package:
    • Create a Notepad file on the desktop, rename it to .cmd.
    • Copy the following code into the file: 
      pushd "%~dp0" 
dir /b %SystemRoot%\servicing\Packages\*Hyper-V*.mum>hyper-v.txt for /f % %i 
in ('findstr /i . hyper-v.txt 2^>null') do  
dism /online /norestart /add-package:"%SystemRoot%\servicing\Packages\%%i" del 
hyper-v.txt Dism /online /enable-feature /featurename:Miscrosoft-Hyper-V-All /LimitAccess /ALL
  3. Run as administrator: Right-click the file and select “Run as administrator”. Wait for the process to complete, then enter “Y” to restart the computer as prompted.
  4. Enable Hyper-V: After restarting, go to “Settings > Apps > Programs and Features > Turn Windows features on or off” to find and enable the Hyper-V option.

Alternative steps:

  1. Run cmd as administrator and execute: 
    bcdedit /copy {default} /d "Windows 10 Without Hyper-V"
bcdedit /set {xxxxx} hypervisorlaunchtype off
    Replace xxxxx with the ID obtained from the first command.
  2. Run msconfig, set the boot timeout to 3–5 seconds or more.
  3. Restart and select “Windows 10 Without Hyper-V” to run VMware. To use Hyper-V, restart and select the default boot option.

4. SM3 Digest Algorithm Result Converts to Garbled String in HarmonyOS?

// SM3Hash
export async function SM3Hash(plainText: string): Promise<string> {
  let mdAlgName = 'SM3'; // Digest algorithm name
  let md = cryptoFramework.createMd(mdAlgName);
  // For small data volumes, you can perform a single update with all data passed in; the interface has no length limit for input parameters
  await md.update({ data: new Uint8Array(buffer.from(plainText, 'utf-8').buffer) });

The uint8ArrayToString method used here is a decryption decoding method. Instead, convert to base64 or a hexadecimal data base64-formatted string:

2. What is the best practice for storing mass image resources in a HarmonyOS project?

Choose based on requirements:

  • base directory: The default directory. 5. Does the HarmonyOS time not have the isAfter attribute? How to judge whether the current time is after the target time?

You can use the getTime() function to convert the date into a timestamp and then subtract it from the current timestamp.
