1. HarmonyOS Bluetooth Key Module?

After setting the setCharacteristicChangeNotification in the Bluetooth key module, the characteristic value change cannot be monitored in the BLECharacteristicChange subscription event.
Steps:

  1. Call the setCharacteristicChangeNotification interface to set the notify property of the characteristic to true.
  2. Call the writeDescriptorValue interface to set the descriptor value of the characteristic to {0x01, 0x00}.
  3. Call the writeCharacteristicValue interface to successfully send 7 - byte data to the master device.
  4. The returned data cannot be monitored in the BLECharacteristicChange callback.

If the device itself supports data return after writing data, it is necessary to check whether the interface method of writeCharacteristicValue is inconsistent with the service of the device that will return data. You can refer to the following: https://gitee.com/openharmony/docs/blob/master/zh - cn/application - dev/reference/apis - connectivity - kit/js - apis - bluetooth - ble.md#blecharacteristic

2. Does DevEco Studio Have a Tool for Viewing the Hierarchical Distribution of Page Elements?

Is there a function that allows you to click on a control or component to see the hierarchical distribution of its UI on the real - device page?

You can check whether ArkUI Inspector meets your needs. Refer to the documentation: https://developer.huawei.com/consumer/cn/doc/harmonyos - guides - V5/ide - arkui - inspector - V5

Developers can use ArkUI Inspector to view the UI display effect of the application on the real - device in DevEco Studio, and quickly analyze and locate the problems of the UI interface by viewing the interface state after multiple operations.

The functions supported by ArkUI Inspector include:

  • View the UI display effect of the application on the device.
  • Export and import UI interface snapshots of the application to view the UI display effect without the device.
  • Select a component in the component tree, and the UI interface will automatically frame the corresponding component, and the property list will display the property information of the current component.
  • Click to select a component in the UI interface, the corresponding component in the component tree will change to the selected state, and the property list will display the property information of the current component.
  • UI component source code jump. After selecting a UI component, click the source code jump button to jump to the source code location.
  • Select Show Component Border in the UI interface to view the display area of all components on the current page.
  • Select a custom component in the component tree, and the property list will display the state variable information and influencing components configured for the current component.

4. How to Configure HarmonyOS to Display Notifications on the Lock - screen Interface and in Banners?

The Huawei push capability is accessed in the docking with the backend, but now I find that the received messages are not displayed on the lock - screen interface, and there is no banner pop - up. The push messages can only be seen in the “More Notifications” of the notification center. How to configure it to display notifications on the lock - screen interface and in banners normally?

  1. The message reminder method is related to the message category. For example: a. The reminder methods for service and communication categories are: lock - screen, ringtone, and vibration. b. The message reminder method for information and marketing categories is silent notification, and the message is only displayed in the notification center. The current developer’s message type should be an information and marketing - type message.
  2. If the sent message is indeed a service and reminder - type message, it can be achieved by applying for self - classification.
  3. Finally, for the reminder methods of lock - screen interface and banner notification, currently, only “IM” and “VOIP” messages under the service and reminder category support lock - screen interface and banner notification.

2. What is the best practice for storing mass image resources in a HarmonyOS project?

The project contains numerous image resources. Does Huawei have a best practice for directory placement? Currently, the project has two directories: base/media and rawfile. Which directory is recommended?

Choose based on requirements:

  • base directory: The default directory. The secondary subdirectory element stores basic elements like strings, colors, and booleans. media and profile store resources such as media, animations, and layouts. Resources here are compiled into binary files with assigned IDs, referenced by specifying the resource type and name.
  • rawfile directory: Supports creating multi-level subdirectories with custom names, allowing free placement of various resources. Resources here are packaged directly without compilation or assigned IDs, referenced by file path and name.

invokeKeyboard(callback?: (uri: string) => void) {
  const context = getContext(this) as common.UIAbilityContext;
  context.startAbilityForResult({
    action: "ohos.want.action.dial",
    parameters: {
      callBundleName: "com.example.mydemo"
    }
  }, (err: BusinessError, data: common.AbilityResult) => {
    if (err && err.code !== 0) {
      console.error(`imageCapture error: ${JSON.stringify(err)}`);
      return;
    }
    console.warn(`imageCapture data: ${JSON.stringify(data)}`);
    let uri = data?.want?.parameters?.resourceUri;
    if (callback && uri) {
      callback(uri.toString());
    }
  });
}

5. Does the HarmonyOS time not have the isAfter attribute? How to judge whether the current time is after the target time?

date.isAfter(new Date(2024, 7, 1)), does the time not have the isAfter attribute? If I want to judge whether the current time is after this time?
You can use the getTime() function to convert the date into a timestamp and then subtract it from the current timestamp. If it is greater than 0, it means that the current time is before 0:00 on July 1, 2024, and if it is less than 0, it is after 0:00. If you want to detect whether it is between 0:00 and 23:59 on July 1, 2024 (that is, on the same day), you can compare the current time with 0:00 on July 1, 2024, and 0:00 on the 2nd respectively to determine the interval where the current time is located.
