1. Performance issues caused by the time-consuming getStringSync and getNumber methods in HarmonyOS?

In many scenarios on the page, the getContext().resourceManager.getStringSync() method is used to retrieve resources from the resource pool for use elsewhere. However, analysis shows that the getStringSync() and getNumber() methods are extremely time-consuming, taking over a hundred milliseconds each time they execute. This significantly lengthens the startup time when they are called during the home page loading.

Solution:
When using the getStringSync(resId: number, …args: Array<string | number>): string method, directly passing resource objects as parameters is time-consuming. Instead, use resource IDs (resourceId) to improve performance. The same applies to the getNumber method.

2. How to detect buffering states and playback freezes in HarmonyOS avplayer?

How can I detect when the avplayer is buffering or frozen due to insufficient buffer? Monitoring bufferingUpdate and checking for media.BufferingInfoType.BUFFERING_START and media.BufferingInfoType.BUFFERING_END does not fully solve the problem. Will these two states switch frequently during normal playback?

Requirements:

  2. Show a loading indicator when scrubbing to an unbuffered position until playback resumes.

Solution:
Monitor the buffered duration (CACHED_DURATION) in the bufferingUpdate callback to detect buffering states and playback freezes.
Reference: https://developer.huawei.com/consumer/cn/doc/harmonyos-references-V2/js-apis-media-0000001427902672-V2#ZH-CN_TOPIC_0000001523488666__avplayer9

3. Which long-chain Socket technology should be selected for HarmonyOS with high stability requirements (currently using TcpSocket)?

You can use the Socket connection capabilities provided by the Network Kit, which supports TCP/UDP/Multicast/TLS protocols for data transmission.

Steps:

  1. Import the required socket modules.
  2. Create a TCPSocketServer instance to return a TCPSocketServer object.
  3. Bind to a local IP address and port, then listen for and accept client TCPSocket connections.
  4. Subscribe to the connect event of TCPSocketServer to monitor client connection status.
  5. After a client connects, a TCPSocketConnection object is returned for communication.
  6. Subscribe to relevant events of TCPSocketConnection to send/receive data.
  7. Manually close the client connection when necessary.
  8. Unsubscribe from all events to release resources.

Reference: https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/socket-connection-V5

5. What are the differences between Online Authentication Kit and User Authentication Kit, and their respective use cases?

Both support fingerprint and face authentication. What are their different application scenarios and security levels?

Answers:

  1. Security Level: Both the User Authentication Kit and FIDO/IFAA Online Authentication Kit provide TEE-level security. Biometric authentication is performed within the Trusted Execution Environment (TEE).
  2. User Authentication Kit:
    • Performs local fingerprint/face authentication.
    • Returns a trusted authentication result for local use cases like login or payment.
      -后续业务逻辑需自行处理认证结果的应用。
  3. FIDO/IFAA Online Authentication:
    • Based on industry-standard protocols defined by alliances (FIDO/IFAA).
    • Integrates public/private keys, biometric data, and server-side verification for passwordless authentication and payment.
    • Requires server-side implementation of FIDO/IFAA-compliant services.
    • A more standardized solution for passwordless authentication.

Summary: Use the User Authentication Kit for local authentication needs, and FIDO/IFAA for cross-platform passwordless solutions requiring server integration.

1. After selecting an image from the album in HarmonyOS, which returns the image URI, how can I convert the image to Base64 based on the URI?

Refer to the following conversion example: 

let packOpts: image.PackingOption = { format: "image/jpeg", quality: 98 };
let imagePackerApi = image.createImagePacker();

const context: Context = getContext(this);
const resourceMgr = context.resourceManager;
let imageBuffer = await resourceMgr.getMediaContent($r('app.media.ic_low'));
const imageSource: image.ImageSource = image.createImageSource(imageBuffer.buffer);
let decodingOptions: image.DecodingOptions = {
  editable: true,
  desiredPixelFormat: 3,
  rotate: 90
}
// Create a pixelMap and perform simple rotation and scaling  

imageSource.createPixelMap(decodingOptions).then((pixelMap: PixelMap) => {

  this.pixelMap = pixelMap
  // To convert pixelmap to Base64, a packing process is required first  
  imagePackerApi.packing(pixelMap, packOpts).then((data: ArrayBuffer) => {
    let base64Str = buffer.from(data).toString('base64')
    let resultBase64Str = "data:image/png;base64," + base64Str
    this.resultBase64Str = resultBase64Str
  }).catch((error: BusinessError) => {
    console.error('Failed to pack the image. And the error is: ' + error);
  })
  pixelMap.release();
})

1. HarmonyOS Bluetooth Key Module?

After setting the setCharacteristicChangeNotification in the Bluetooth key module, the characteristic value change cannot be monitored in the BLECharacteristicChange subscription event.
Steps:

  1. Call the setCharacteristicChangeNotification interface to set the notify property of the characteristic to true.
  2. Call the writeDescriptorValue interface to set the descriptor value of the characteristic to {0x01, 0x00}.
  3. Call the writeCharacteristicValue interface to successfully send 7 - byte data to the master device.
  4. The returned data cannot be monitored in the BLECharacteristicChange callback.

If the device itself supports data return after writing data, it is necessary to check whether the interface method of writeCharacteristicValue is inconsistent with the service of the device that will return data. You can refer to the following: https://gitee.com/openharmony/docs/blob/master/zh - cn/application - dev/reference/apis - connectivity - kit/js - apis - bluetooth - ble.md#blecharacteristic

  2. If the sent message is indeed a service and reminder - type message, it can be achieved by applying for self - classification.
  3. Finally, for the reminder methods of lock - screen interface and banner notification, currently, only “IM” and “VOIP” messages under the service and reminder category support lock - screen interface and banner notification.

2. What is the best practice for storing mass image resources in a HarmonyOS project?

The project contains numerous image resources. Does Huawei have a best practice for directory placement? Currently, the project has two directories: base/media and rawfile. Which directory is recommended?

Choose based on requirements:

  • base directory: The default directory. The secondary subdirectory element stores basic elements like strings, colors, and booleans. media and profile store resources such as media, animations, and layouts. Resources here are compiled into binary files with assigned IDs, referenced by specifying the resource type and name.
  • rawfile directory: Supports creating multi-level subdirectories with custom names, allowing free placement of various resources. Resources here are packaged directly without compilation or assigned IDs, referenced by file path and name.

3. How does an H5 page in HarmonyOS invoke a native custom keyboard?

Implement a JSBridge communication solution by encapsulating javaScriptProxy and runJavaScript, suitable for H5 to invoke native-side functions.
Reference: https://gitee.com/harmonyos/codelabs/tree/master/SelectContact

Demo reference (Web component): 

.onShowFileSelector((event) => {
  console.warn('…MyFileUploader onShowFileSelector invoked');

  console.warn('…invoke keyboard');
  this.invokeKeyboard(uri => {
    console.warn('…uri: ' + uri);
    event?.result.handleFileList([uri]);
  });
  return true;
})

invokeKeyboard(callback?: (uri: string) => void) {
  const context = getContext(this) as common.UIAbilityContext;
  context.startAbilityForResult({
    action: "ohos.want.action.dial",
    parameters: {
      callBundleName: "com.example.mydemo"
    }
  }, (err: BusinessError, data: common.AbilityResult) => {
    if (err && err.code !== 0) {
      console.error(`imageCapture error: ${JSON.stringify(err)}`);
      return;
    }
    console.warn(`imageCapture data: ${JSON.stringify(data)}`);
    let uri = data?.want?.parameters?.resourceUri;
    if (callback && uri) {
      callback(uri.toString());
    }
  });
}

Combined with the onShowFileSelector event of the Web component, the system camera can be launched, and the result is returned to the H5 input tag, automatically triggering its onchange event.
