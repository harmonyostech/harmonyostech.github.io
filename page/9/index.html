1. Does HarmonyOS Webview support proxy settings?

Use the Web component’s onInterceptRequest to intercept and proxy requests. For details, refer to the documentation: https://developer.huawei.com/consumer/cn/doc/harmonyos-references-V5/ts-basic-components-web-V5#ZH-CN_TOPIC_0000001930757269__oninterceptrequest9

onInterceptRequest(callback: Callback<OnInterceptRequestEvent, WebResourceResponse>)
This callback triggers before the Web component loads a URL, used to intercept the URL and return response data.
System Capability: SystemCapability.Web.Webview.Core
Parameters:

Parameter Name Type Required Description
callback Callback Yes Triggers before the Web component loads a URL.

Returns WebResourceResponse. If response data is returned, it loads based on the data; if null, it loads as original.

Example: 

// xxx.ets
import { webview } from '@kit.ArkWeb';

@Entry
@Component
struct WebComponent {
  controller: webview.WebviewController = new webview.WebviewController();
  responseWeb: WebResourceResponse = new WebResourceResponse();
  heads: Header[] = new Array();
  webData: string = "<!DOCTYPE html>\n" +
    "<html>\n" +
    "<head>\n" +
    "<title>intercept test</title>\n" +
    "</head>\n" +
    "<body>\n" +
    "<h1>intercept test</h1>\n" +
    "</body>\n" +
    "</html>";

  build() {
    Column() {
      Web({ src: 'www.example.com', controller: this.controller })
        .onInterceptRequest((event) => {
          if (event) {
            console.log('url:' + event.request.getRequestUrl());
          }
          let head1: Header = {
            headerKey: "Connection",
            headerValue: "keep-alive"
          }
          let head2: Header = {
            headerKey: "Cache-Control",
            headerValue: "no-cache"
          }
          // Append new elements to the end of the array and return the new length.
          let length = this.heads.push(head1);
          length = this.heads.push(head2);
          console.log('The response header result length is :' + length);
          const promise: Promise<String> = new Promise((resolve: Function, reject: Function) => {
            this.responseWeb.setResponseHeader(this.heads);
            this.responseWeb.setResponseData(this.webData);
            this.responseWeb.setResponseEncoding('utf-8');
            this.responseWeb.setResponseMimeType('text/html');
            this.responseWeb.setResponseCode(200);
            this.responseWeb.setReasonMessage('OK');
            resolve("success");
          })
          promise.then(() => {
            console.log("prepare response ready");
            this.responseWeb.setResponseIsReady(true);
          })
          this.responseWeb.setResponseIsReady(false);
          return this.responseWeb;
        })
    }
  }
}

2. Issues with drawing using OffscreenCanvasRenderingContext2D in HarmonyOS

How to add rounded corners to a drawn rectangle (original requirement: rectangle with 8px rounded corners).

Using fillRect cannot create rounded corners as it draws right-angled rectangles. The solution is to use moveTo, arcTo, lineTo, and stroke to draw the rounded rectangle outline, then fill the color. Here’s a simplified example for one corner: 

@Entry
@Component
struct FillStyleExample {
  private settings: RenderingContextSettings = new RenderingContextSettings(true)
  private context: CanvasRenderingContext2D = new CanvasRenderingContext2D(this.settings)
  private offCanvas: OffscreenCanvas = new OffscreenCanvas(600, 600)

  build() {
    Flex({ direction: FlexDirection.Column, alignItems: ItemAlign.Center, justifyContent: FlexAlign.Center }) {
      Canvas(this.context)
        .width('100%')
        .height('100%')
        .backgroundColor('#ffff00')
        .onReady(() => {
          let offContext = this.offCanvas.getContext("2d", this.settings)
          offContext.fillStyle = 0x0000FF
          offContext.moveTo(100, 20)
          offContext.arcTo(300, 20, 300, 70, 8)
          offContext.lineTo(300, 80)
          offContext.stroke()
          // offContext.fill()
          let image = this.offCanvas.transferToImageBitmap()
          this.context.transferFromImageBitmap(image)
        })
    }
    .width('100%')
    .height('100%')
  }
}

2. HarmonyOS router.back is not functional.

Considerations for named routing:

  • Named routing is not limited to module types; it works in shared packages (hsp, har) and in entry/feature modules.
  • Named routing currently requires importing the corresponding component; it will not work without import("module").
  • The @Entry({routeName}) annotation is similar to @Entry, implementing pseudo-page definition in har. routeName corresponds to the name parameter in the pushNameRoute interface.
  • pushNameRoute and pushUrl are unrelated.
  • Components without @Entry cannot be used as pages; only @Entry-decorated components can serve as page entries.

1. HarmonyOS: Column layout containing a List?

When a Column layout includes both normal components and a List, the last item of the List cannot be fully displayed. How to solve this?

The most likely cause is that in a non-Flex linear layout container, the layout constraints of child components are affected by properties like layoutWeight, flexShrink, or flexGrow. If other components and the List have the same layout constraints, the List uses maxHeight to obtain its real height (equal to the Column’s height). Combined with the height occupation of other components, the last part of the List items cannot be fully displayed on the screen. Setting layoutWeight on the List allows the Column to allocate the remaining space to the List, ensuring complete display.

Example code: If the List does not set .layoutWeight(100), the last item of the List cannot be fully displayed.

3. HarmonyOS: The touchstart event does not respond after opening a page in WebView.

The touchstart event conflicts with the click event. When adapting a completed PC site for mobile devices, using mobile-specific event systems (e.g., touchstart instead of click) can improve response speed. Mobile browsers delay the click event by 300ms to distinguish between single and double clicks, while touchstart triggers instantly when the user touches the screen.

However, when adapting a PC site for mobile devices, simply replacing click with touchstart is insufficient because PCs cannot recognize touchstart. Binding both events to an element causes conflicts on mobile devices, as both events trigger sequentially (with click delayed by ~300ms), executing callbacks twice.

Solutions:

  1. Use preventDefault: 

    const Button = document.getElementById("targetButton")

Button.addEventListener("touchstart", e => {
    e.preventDefault()
    console.log("touchstart event!")
})

Button.addEventListener("click", e => {
    e.preventDefault()
    console.log("click event!")
})

    Simulating a mobile device in the browser will only log touchstart event!, blocking the click event.

  2. Bind events based on feature detection: 

    const Button = document.getElementById("targetButton")

const clickEvent = (function() {
  if ('ontouchstart' in document.documentElement === true)
    return 'touchstart';
  else
    return 'click';
})();

Button.addEventListener(clickEvent, e => {
  console.log("things happened!")
})
