1. Why can’t I read file content or does the program crash when operating files according to the documentation?

When writing to or reading from a file as per the documentation, the content cannot be read, or the program crashes.

E C01401/Bufferqueue: … width or height is 0 … Alloc failed …


**Code context**:  
Following the Surface mode in https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/video-encoding-V5, using OpenGL buffer mode:  
1. Obtain `nativeWindow` via `OH_VideoEncoder_GetSurface`.  
2. Create an EGL surface with `eglCreateWindowSurface`.  
3. Swap buffers with the camera texture, but encounter BufferQueue errors.  

**Possible causes**:  
- Width or height parameters are not properly initialized (0 values).  
- Incompatible EGL context or surface configuration.  

**Solution**: Ensure valid dimensions (width/height > 0) when configuring the encoder and EGL surface.  


## 5. Error "No app supports this file type" when jumping to WLAN settings without network  
**Original code** (no longer working):  
```typescript
{
  bundleName: 'com.example.xxx.settings',
  abilityName: 'com.example.xxx.settings.MainAbility',
  uri: 'application_info_entry'
}

Solution:
Update the URI to wifi_entry: 

{
  bundleName: 'com.ohos.settings',
  abilityName: 'com.ohos.settings.MainAbility',
  uri: 'wifi_entry'
}

Reason: Implicit URI jumps to settings pages have been updated. Use the dedicated wifi_entry URI for WLAN settings.
