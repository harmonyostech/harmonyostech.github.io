2. Can’t Create Folders in resources->base->media?

In the project, the image path resources->base->media does not allow creating folders, so all images are placed at the same level, making it inconvenient to find images. Is there any other solution?

Directories for storing images under the resources directory include resources/*/media and resources/rawfile. You can use rawfile to create folders for placing images.

  • rawfile supports creating multi-level subdirectories with custom names, allowing free placement of various resource files.

Notes:

  1. rawfile does not match different resources based on the device environment.
  2. Source files in rawfile are directly packaged into the app without compilation or assigned resource IDs, referenced by file path and name.

3. How to Dynamically Set versionName and VersionCode with hvigor?

Referencing our previous version definition, versionName has 4 segments: the first 3 segments are manually configured, and the 4th segment is obtained by executing git rev-list --count HEAD. The VersionCode is also dynamically generated based on the first 3 segments of versionName.

How to implement dynamic generation of versionName and VersionCode in HarmonyOS native or hvigor?

Please refer to the version normalization command (versionNormalize):
https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/packing-tool-V5#%E7%89%88%E6%9C%AC%E5%BD%92%E4%B8%80%E6%8C%87%E4%BB%A4versionnormalize

4. SM3 Digest Algorithm Result Converts to Garbled String in HarmonyOS?

// SM3Hash
export async function SM3Hash(plainText: string): Promise<string> {
  let mdAlgName = 'SM3'; // Digest algorithm name
  let md = cryptoFramework.createMd(mdAlgName);
  // For small data volumes, you can perform a single update with all data passed in; the interface has no length limit for input parameters
  await md.update({ data: new Uint8Array(buffer.from(plainText, 'utf-8').buffer) });
  let mdResult = await md.digest();
  console.info('Md result:' + mdResult.data);
  return uint8ArrayToString(mdResult.data);
}

The uint8ArrayToString method used here is a decryption decoding method. Instead, convert to base64 or a hexadecimal data base64-formatted string:

  • Base64 string: new util.Base64Helper().encodeToStringSync(str)
  • Hexadecimal string: buffer.from(data).toString('hex')

5. Error When Storing JSON-Format Strings in Preferences?

When storing JSON-format strings in Preferences, an error occurs:
Whether converted to: 

let uInt8Array = new util.TextEncoder().encodeInto(value);
this.preferences.putSync("asd", uInt8Array)

Or directly: 

this.preferences.putSync("asd", value)

The error is: Parameter error. The type of ‘value’ must be ValueType.

Cause: The value is too long. The maximum length limit for preferences value is 8192 bytes. Consider using a KV database or RDB for storage.
