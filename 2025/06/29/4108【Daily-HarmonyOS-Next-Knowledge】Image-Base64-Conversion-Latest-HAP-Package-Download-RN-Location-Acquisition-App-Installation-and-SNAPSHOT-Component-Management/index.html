1. After selecting an image from the album in HarmonyOS, which returns the image URI, how can I convert the image to Base64 based on the URI?

Refer to the following conversion example: 

let packOpts: image.PackingOption = { format: "image/jpeg", quality: 98 };
let imagePackerApi = image.createImagePacker();

const context: Context = getContext(this);
const resourceMgr = context.resourceManager;
let imageBuffer = await resourceMgr.getMediaContent($r('app.media.ic_low'));
const imageSource: image.ImageSource = image.createImageSource(imageBuffer.buffer);
let decodingOptions: image.DecodingOptions = {
  editable: true,
  desiredPixelFormat: 3,
  rotate: 90
}
// Create a pixelMap and perform simple rotation and scaling  

imageSource.createPixelMap(decodingOptions).then((pixelMap: PixelMap) => {

  this.pixelMap = pixelMap
  // To convert pixelmap to Base64, a packing process is required first  
  imagePackerApi.packing(pixelMap, packOpts).then((data: ArrayBuffer) => {
    let base64Str = buffer.from(data).toString('base64')
    let resultBase64Str = "data:image/png;base64," + base64Str
    this.resultBase64Str = resultBase64Str
  }).catch((error: BusinessError) => {
    console.error('Failed to pack the image. And the error is: ' + error);
  })
  pixelMap.release();
})
// Key tip: Converting pixelmap to Base64 cannot be done directly. A packing process must be added first. The following method is an incorrect example, and the converted Base64 string will be invalid.

2. How to install the latest HAP package downloaded from the server in HarmonyOS?

Our home page has a version update function. Can we place our APP (HAP) package on the server, call the system API to install the new version after downloading it on the home page? How to install the downloaded HAP, does it require permissions, and are there version update examples?

Except for local debugging, the only way to install a HAP package is through the AppGallery. Installing via API is not supported.
The AppGallery provides update and upgrade detection capabilities. Refer to the link: https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/store-update-V5

3. What methods does RN have to obtain detailed location information, such as province, city, district, and street?

RN does not have the capability to obtain location information. It is recommended to implement it through TurboModule via the OS. The documentation links for the OS to obtain location information are as follows:

  1. Get the current latitude and longitude: https://developer.huawei.com/consumer/cn/doc/harmonyos-references-V5/js-apis-geolocationmanager-V5#geolocationmanagergetcurrentlocation
  2. Obtain province, city, district, and street information based on latitude and longitude: https://developer.huawei.com/consumer/cn/doc/harmonyos-references-V5/js-apis-geolocationmanager-V5#geolocationmanagergetaddressesfromlocation

4. How to install an app in HarmonyOS?

Which API is called to install the app after it is successfully downloaded in the app upgrade function?

App installation uses @ohos.bundle.installer.

5. About SNAPSHOT component management and usage?

  1. If our version is 1.0.0-SNAPSHOT, should we directly use publish + tag to release the component to the private repository, or use the dist-tags command for management?
  2. If the component version we release is 1.0.0-SNAPSHOT, do we need to configure the version as “name”: “1.0.0-SNAPSHOT:snapshot” when using it, and can this be done in override?
  3. If the component version is 1.0.0 and the --tag command is used during release, will it affect the formally released 1.0.0 version?
  4. If the same version releases multiple SNAPSHOT packages, will the local version be updated via ohpm install? Or should ohpm update be used?

Answers:

  1. dist-tags can only be used to add tags, not for releasing. Use publish to push the package. Use publish --tag to specify the tag for the release version. The specified tag must not exist; otherwise, the release will fail. If the tag already exists, use dist-tag update to re-specify the version corresponding to the tag.
  2. The usage is as follows, and it supports overrides:
"dependencies": {    
  "<dependency-name>": "tag:<tag-name>"  
}
  1. It will not affect. The same package and version can only be released once, and the same package tag cannot be repeated. If the specified tag already exists during publish, the release will fail.
  2. install will not update. Use ohpm update for updates.
