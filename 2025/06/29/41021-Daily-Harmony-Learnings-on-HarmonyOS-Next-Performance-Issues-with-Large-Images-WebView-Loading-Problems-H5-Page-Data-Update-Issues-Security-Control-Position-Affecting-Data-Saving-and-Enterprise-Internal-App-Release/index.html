**Requirement**: Security controls cannot be placed at the screen edge; even slight overflow causes failure.  
Reference: https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/security-component-overview-V5#ZH-CN_TOPIC_0000001884756418__%E7%BA%A6%E6%9D%9F%E4%B8%8E%E9%99%90%E5%88%B6

**Operation mechanism**:  
1. Developers integrate security controls in ETS files, which are parsed by the JS engine to generate controls in the ArkUI framework.  
2. Security controls register information with the security control management service, which checks legitimacy.  
3. User click events are dispatched to security controls.  
4. Security controls report click events to the management service.  
5. The management service invokes the permission management service for temporary authorization based on control types.  
6. Upon successful authorization, security controls callback `OnClick` to notify the app layer.  
7. The app performs privileged operations (e.g., getting location, reading clipboard, creating files in the media library).  
8. Different control types have varying permission usage and authorization validity (refer to specific development guides).  
9. Corresponding services obtain authorization results from the permission or security control management service and return authentication results.
