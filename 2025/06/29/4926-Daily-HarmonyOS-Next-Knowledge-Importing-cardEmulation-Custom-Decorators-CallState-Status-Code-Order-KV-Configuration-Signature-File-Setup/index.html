1. Failed to import cardEmulation in HarmonyOS?

Importing cardEmulation fails in the index.ets file of the entry module.

Solution: Add the SystemCapability in src/main/syscap.json (create this file manually) with the following content: 

{
  "devices": {
    "general": [
      // Must match the device type in module.json5
      "phone"
    ]
  },
  "development": {
    "addedSysCaps": [
      "SystemCapability.Communication.NFC.CardEmulation"
    ]
  }
}

2. How to use custom decorator ClassDecorator?

ArkTS supports TypeScript decorator syntax prior to TS5.0. For decorator definitions and runtime behavior, refer to the TS official documentation: https://www.typescriptlang.org/docs/handbook/decorators.html. Note that if defining decorators in .ets files, they must comply with ArkTS syntax rules (e.g., no use of any).

Simple examples for class, property, method, and parameter decorators: 

function TestClassDecorator(target: Function) {}
function TestMemberDecorator(target: testClass, memberName: String) {}
function TestFunDecorator(target: testClass, propertyName: String, descriptor: PropertyDescriptor) {}
function TestArgDecorator(target: Function, methodName: String, paramIndex: Number) {}

@TestClassDecorator
class testClass {
  @TestMemberDecorator
  count: number = 123;

  @TestFunDecorator
  TestFun(@TestArgDecorator param: string) {}
}

3. What is the order of CallState status codes when a user makes a call in HarmonyOS?

Status code changes for three scenarios when a user dials (outgoing call), including rejection, answer, or no answer:

  • Active call rejected by the other party / Caller hangs up during ringing: 2 -> 0
  • Other party answers: 2 -> 2 (status code remains 2 during ringing)

4. How to create and read key-value custom configurations in HarmonyOS?

Requirement: Store key-value custom configurations accessible to all modules in the app directory, with values of flexible types (string, number, boolean). The goal is to read values directly by key without parsing the entire file.

1. Create a JSON file under resources/base/element/ (e.g., boolean.json):

{
  "boolean": [
    {
      "name": "module_desc",
      "value": true
    },
    {
      "name": "EntryAbility_desc",
      "value": false
    },
    {
      "name": "EntryAbility_label",
      "value": false
    }
  ]
}

2. Access via resource management API (getBoolean):

import { common } from '@kit.AbilityKit';

@Entry
@Component
struct Index {
  @State message: string = 'Hello World';
  private context = getContext(this) as common.UIAbilityContext;

  build() {
    RelativeContainer() {
      Text(this.message)
        .id('HelloWorld')
        .fontSize(50)
        .fontWeight(FontWeight.Bold)
        .onClick(() => {
          let aa: boolean = this.context.resourceManager.getBoolean($r('app.boolean.EntryAbility_desc').id);
          console.log('lujun EntryAbility_desc value is ' + aa);
        })
        .alignRules({
          center: { anchor: '__container__', align: VerticalAlign.Center },
          middle: { anchor: '__container__', align: HorizontalAlign.Center }
        })
    }
    .height('100%')
    .width('100%')
  }
}

5. Issue: Signature file not supported during HarmonyOS signing

Problem: When packaging a HarmonyOS app, a “signature file not supported” error occurs, even though the signature file was generated via commands.

Scenario description:

Generating a signature file with the command: 

keytool -genkeypair -alias "zofund" -keyalg EC -sigalg SHA256withECDSA -dname "C=CN,O=zofund,OU=zofund,CN=zofund"  -keystore /Users/lidaofu/DevEcoStudioProjects\debug.p12 -storetype pkcs12 -validity 9125 -storepass *** -keypass ****

Note: The profile for signing should be a .p7b file.

Manual signing process:

  1. Generate a CSR file:
    In the editor, go to Build > Generate Key and CSR.
  2. Select “Use existing p12” (or create a new one), fill in Alias, Password, etc., and remember the credentials.
  3. Click Finish to complete CSR creation.

Apply for a debug signature certificate:

  1. Log in to AppGallery Connect, go to Users & Access.
  2. In Certificate Management on the left, click Add Certificate, upload the generated CSR, and download the certificate for future use.
  3. In Device Management on the left, add corresponding devices.

Apply for a debug profile:

  1. Log in to AppGallery Connect, go to My Projects.
  2. Find the project, navigate to Manage HAP Provision Profile.
  3. Manage the profile as needed.

Related documentation: https://developer.huawei.com/consumer/cn/doc/app/agc-help-debug-app-0000001914423098
