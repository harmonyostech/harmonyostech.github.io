1. What UI automation tools for HarmonyOS can be used directly?

For the automation testing framework, refer to: https://developer.huawei.com/consumer/cn/doc/harmonyos-guides-V5/arkxtest-guidelines-V5

2. Does the DevEco low-code platform support direct invocation of closed-source SDK capabilities?

Currently, it does not support this. The current low-code platform can only be used for UI design. It does not support the capabilities of third-party SDKs. After completing development in the low-code platform and generating the source code package, you need to use ArkTS to code and invoke the capabilities manually based on the source code.
Meanwhile, the third-party SDK must be a HarmonyOS-compatible SDK.

3. Does HarmonyOS have an interface to obtain the screen brightness?

Currently, the official documentation recommends obtaining the screen brightness by first acquiring the window and then using windowProperties. However, this method asynchronously retrieves the screen brightness. Can an interface for synchronously obtaining the screen brightness be developed? 

// Get the system screen brightness value  
return Settings.System.getInt(contentResolver, Settings.System.SCREEN_BRIGHTNESS)

For obtaining the screen brightness, refer to: https://developer.huawei.com/consumer/cn/doc/harmonyos-references-V5/js-apis-settings-V5

4. How to compile and use mbedtls in HarmonyOS?

When developing native applications, mbedtls is required. There was a plan to directly use a precompiled static library for the v8a architecture, but an error occurred when compiling the HarmonyOS SDK.
You need to use cross-compilation as described in https://gitee.com/openharmony-sig/tpc_c_cplusplus/blob/master/lycium/doc/ohos_use_sdk/OHOS_SDK-Usage.md, and then refer to https://gitee.com/openharmony-sig/tpc_c_cplusplus/blob/master/lycium/doc/app_calls_third_lib.md for integration.

5. When connecting a test device via USB, some data cables can only charge the device and cannot be recognized by DevEco. What should be done?

  1. Ensure that the data cable is a genuine Huawei mobile phone-specific cable purchased through formal channels.
  2. For data cables that can only charge but cannot be recognized by DevEco, confirm whether the current DevEco version is 4.1.1.500 or higher.
  3. After connecting the problematic data cable to the computer, open the CMD command line and execute the following command: hdc list targets to check for any output. If there is no output, add hdc to the environment variables. You can perform a global search for hdc.exe and then execute the above command.
  4. If the output in step 3 is [Empty], it indicates that the data cable is faulty and needs to be replaced with a normal one.
